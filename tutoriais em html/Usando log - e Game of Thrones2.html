Transformao por logaritmo

Uma soluo rpida (e talvez grosseira) para o nosso problema tacar log nos nmeros! Veja bem, quando calculamos o log de um conjunto de nmeros, ns mantemos a ordem deles (ou seja, quem era maior continua maior, quem era menor continua menor), mas achatamos as diferenas. Veja o exemplinho a seguir:

log10(exemplo)
## [1] 0 1 2 3 4

Viu s? A diferena entre os nmeros ficou pequena, mas a ordem no se perdeu. Se fizermos este mesmo tipo de transformao nos nossos dados, o resultado pode ser bem til. Vamos l?

Veja que legal como ficou! Continuamos a ver que a diferena da nossa sp14 para as demais enorme, mas agora enxergamos melhor as diferenas entre as demais. Grfico com escala logaritmizada

Uma outra maneira de trabalhar com este problema mostrar os nmeros reais, mas logaritmizar a escala do eixo y. Desta forma, o grfico vai sair quase a mesma coisa que o anterior, mas a escala dos nmeros no eixo vai ser outra. olha s como :

grfico2<-barplot(somas, las=2, ylim=c(1, 5000), ylab="Densidade (log)", xlab="Espcies", log="y")

Aqui bastou um novo argumento: log="y", que foi o responsvel pela nova escala. Note que eu mexi no ylim aqui, para melhorar a visualizao do eixo (de novo, os valores ideiais foram encontrados na base da tentativa e erro mesmo).

Bonito, n? Veja que a escala muda de maneira diferente, mas os nmeros so reais, o que pode ser um meio termo bem legal para se mostrar dados assim.
