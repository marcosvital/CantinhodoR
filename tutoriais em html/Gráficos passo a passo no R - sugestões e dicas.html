Do que vamos precisar - um pouco sobre os conjuntos de dados no R

Vou usar uma abordagem diferente desta vez. No lugar de fornecer dados em um arquivo ou na internet, que tal trabalharmos com dados que j esto no R? Pois , voc pode nem saber disso ainda, mas cada pacote do R tem um ou mais conjuntos de dados, que podem ser invocados com a funo data.

Se quiser, o R pode listar para voc os conjuntos de dados disponveis:

data()

Legal, n? O resultado no aparece aqui no nosso markdown porque ele no vai para o console, e sim para uma janela.

Agora, se quer saber sobre todos os dados, incluindo os pacotes que voc tem mas que no esto carregados, faa isso:

data(package = .packages(all.available = TRUE))

Mas ateno: vai demorar um pouquinho pra carregar, e a lista pode ser bem longa se voc tiver muitos pacotes, ok?

Finalmente, se voc quiser conhecer os conjuntos de dados de um pacote especfico, faa assim:

data(package = "vegan")

(Claro, o exemplo acima s vai funcionar se voc tiver o pacote vegan no seu computador, ok?)

Ah sim, e uma ltima informao importante: se quiser, voc pode chamar o help de qualquer conjunto de dados desses, que o R vai te dar mais informaes a respeito. Olha que bacana:

?InsectSprays
## starting httpd help server ... done

Legal, n?

Conhecendo o famoso Iris dataset

Hein? Famoso?! Pois , acredita que um conjunto de dados pode ser famoso?

Acontece que um dos pais da estatstica (sim, essa mesma que andamos usando por aqui), Sir Ronald Fisher publicou um estudo em 1936, no qual explorava um pouco como os mtodos estatsticos poderiam ser aplicados para se estudar taxonomia. Para apresentar os mtodos que queria, Fisher pegou emprestado um conjunto de dados de Edgar Anderson, um botnico, e mais tarde os dados comearam a ser usados amplamente em testes da anlise discriminante que o Fisher desenvolveu. Atualmente ele acabou se tornando um conjunto de dados conhecido mesmo, e tem at pgina na wikipedia, olha s!

Enfim, como no poderia deixar de ser, estes dados esto hoje disponveis para quem quiser usar no R. E o bacana que ele um timo conjunto de dados para se brincar: d pra usar como exemplo de um monte de coisas, e se voc sair por a olhando tutoriais do R na internet, invevitavelmente vai topar com um que usa o Iris Dataset.

Bom, voc agora j sabe o que precisa sobre os dados, ento vamos l? Carregar fcil, fcil:

data(iris)
attach(iris)

E recomendo chamar o help pra ver a explicao sobre as variveis:

?iris

Pronto. Bora trabalhar com grficos?

E a, como o seu lance com os grficos?

Re-aproveitando aquela velha piadinha fraca que ns professores adoramos, o meu modo de trabalhar com grficos no R , como diria o estripador, ir por partes. Voc comea com um comando bem bsico, que cria o grfico sem praticamente nenhuma especificao (nomes de eixos, tipos de smbolos, cores e tal), e depois vai adicionando cada modificao desejada uma a uma.

O que tem de legal nisso, voc pergunta? Bom, o bacana nisso que voc diminui muito a chance de ter um erro difcil de identificar. O que acontece que um comando de grfico no R pode ficar bem longo com muita facilidade, pois cada detalhe modificado um argumento a mais na funo. A, se voc escreve tudo de uma vez, pode dar tudo errado e voc corre o risco de no saber onde o erro est. Sacou?

Ao fazer o grfico passo a passo, argumento por argumento, voc vai perceber imediatamente se algo sair errado. E assim a vida fica bem mais fcil, pois voc ir se frustrar menos e aprender mais.

Ento vamos fazer um exemplo disso a?

Vamos l, tendo carregado o nosso Iris Dataset, vamos montar um grfico assim, passo a passo. Assim demonstro o mtodo que estou sugerindo para lidar com grficos no R, e de quebra vocs podem aprender alguma coisa legal que no conheciam antes. Vamos?

plot(Petal.Length, Petal.Width)

Beleza, grafico bsico sem nada demais. Vamos comear mudando a orientao dos nmeros no eixo y, que esto deitados de lado. Basta usar o argumento las pra alterar isso, assim:

plot(Petal.Length, Petal.Width, las=1)

Legal, melhorou, n? Este foi o nosso primeiro passo. A proposta adicionar argumentos um a um, sem atropelo. Caso voc erre qualquer coisa em um argumento, vai saber que o erro foi na ltima modificao que fez.

Vamos mudar o nome do eixo y, com o argumento ylab:

plot(Petal.Length, Petal.Width, las=1, ylab="Largura das ptalas")

No se esquea que o nome deve ficar entre aspas, ok? Repetindo o mesmo, agora com o eixo x, usando xlab:

plot(Petal.Length, Petal.Width, las=1, ylab="Largura das ptalas",
     xlab="Comprimento das ptalas")

Note que desta vez eu continuei o comando na linha abaixo. Voc pode fazer isso sempre que quiser, bastando dar um enter aps uma vrgula, e continuar na linha seguinte. Gosto disso, pois evita que o comando fique muito comprido e difcil de se ver de uma vez.

Agora vou mudar o smbolo do grfico, usando pch:

plot(Petal.Length, Petal.Width, las=1, ylab="Largura das ptalas",
     xlab="Comprimento das ptalas", pch=16)

Note que eu sempre dou um espao entre os argumentos, aps a vrgula. Ele opcional, voc no precisa inclu-lo no seu comando. Mas gosto de fazer isso sempre, pois torna os comandos mais legveis.

Agora vamos mudar a cor dos pontos, usando uma cor para cada espcie, usando o argumento col:

plot(Petal.Length, Petal.Width, las=1, ylab="Largura das ptalas",
     xlab="Comprimento das ptalas", pch=16, col=Species)

Legal, n?

Mas, da forma como eu fiz, o R escolheu sozinho as cores. Vamos definir as cores ns mesmos? Vai ser bem melhor.

Primeiro, cheque a ordem das espcies assim:

levels(Species)
## [1] "setosa"     "versicolor" "virginica"

Beleza, agora sabemos a ordem. Vamos ento escolher as cores pra cada espcie:

cores<-c("red", "green4", "blue")

Pera, porque voc criou um objeto, Marcos? que eu acho que vai ficar mais fcil de escrever o resto, pois podemos usar este objeto depois de novo, na hora de montarmos nossa legenda. Faz parte do nosso processo de passo a passo.

Vamos colocar as cores no grfico:

plot(Petal.Length, Petal.Width, las=1, ylab="Largura das ptalas",
     xlab="Comprimento das ptalas", pch=16, col=cores[Species])

Massa.

Mas claro que falta uma legenda, no ? Ento vamos usar o mesmo processo, fazer passo a passo para no nos atropelarmos, ok? Vamos comear:

plot(Petal.Length, Petal.Width, las=1, ylab="Largura das ptalas",
     xlab="Comprimento das ptalas", pch=16, col=cores[Species])

legend("topleft", legend=levels(Species))

Beleza, agora vamos continuar, incluindo os smbolos:

plot(Petal.Length, Petal.Width, las=1, ylab="Largura das ptalas",
     xlab="Comprimento das ptalas", pch=16, col=cores[Species])

legend("topleft", legend=levels(Species), pch=16)

Pronto, acho que s falta colocar as cores, no ? Vamos:

plot(Petal.Length, Petal.Width, las=1, ylab="Largura das ptalas",
     xlab="Comprimento das ptalas", pch=16, col=cores[Species])

legend("topleft", legend=levels(Species), pch=16, col=cores)

A, ficou bacana, no foi?
