Passo a passo para criar um cluster:

Etapa 0: se necessrio, padronize os seus dados

dados.amb<-dados[,3:6] #Separei os dados de interesse
dados.amb.pad<-scale(dados.amb) #Padronizei os dados

A novidade aqui a funo scale que padroniza os dados (subtrai da mdia e divide pelo desvio padro).

Etapa 1: crie sua matriz de distncia

amb.dist<-dist(dados.amb.pad, method="euclid")

Aqui usamos a funo dist, e escolhemos a distncia euclidiana com o argumento method

Etapa 2: crie o seu cluster

amb.cluster<-hclust(amb.dist, method="average")

Usamos a funo hclust, e definimos o mtodo de UPGMA pelo argumento method

Etapa 3: calcule o coeficiente de correlao cofentica

matriz.cof<-cophenetic(amb.cluster)  #Cria a matriz coefentica
cor(amb.dist, matriz.cof)               #Calcula o coeficiente
## [1] 0.7388642

Aqui usamos a funo cophenetic para extrair a matriz cofentica, e a funo cor para calcular a correlao de pearson.

O coeficiente apresentou um valor adequado, ento vamos l plotar o nosso cluster!

Plotando o nosso cluster

Existem diversas maneiras de se plotar um cluster no R. Vamos explorar apenas algumas, ok?

Rpido!

O mais rpido e simples :

plot(amb.cluster)

No la muito bonito, mas quebra galho para uma visualizao rpida.

Voc pode usar o argumento hang para que ele estique os ramos e deixe os nomes lado a lado, assim:

plot(amb.cluster, hang=-1)

De lado

Voc pode colocar o cluster na horizontal convertendo ele em outra categoria de objeto. So duas alternativas: voc pode usar a funo as.phylo do pacote ape; ou pode usar a as.dendrogram.

Usando a primeira:

library(ape)

plot(as.phylo(amb.cluster))

E a segunda (neste caso precisamos do argumento horiz para deix-lo na horizontal:

plot(as.dendrogram(amb.cluster), horiz=T) #horiz=T faz na horizontal

Mas nos dois casos uma coisa no ficou legal: os nomes ficaram espremidos

Isto pode ser resolvido de uma maneira simples: aumentando o tamanho da janela do grfico. Vamos l:

windows(width=8, height=8, rescale="fixed")
plot(as.dendrogram(amb.cluster), horiz=T)

Aqui temos novidades:

  • a funo windows abre uma janela para grfico com o tamanho que determinarmos;
  • os argumentos width e heighdeterminam a largura e altura da janela, respectivamente;
  • o argumento rescale="fixed" faz com que a figura no mude de tamanho quando mexemos na janela;

Alterando nomes

Uma das coisas legais do R que podemos mexer aqui e ali e encontrar maneiras de solucionar problemas, representar coisas e tudo mais sem ter que voltar atrs, mexer nos dados originais e coisas do tipo.

Imagine, por exemplo, que voc queira alterar os nomes das unidades amostrais para identific-las como sendo pertencentes aos dois tipos de mata descritos na varivel ambiente. Como fazer isso sem sair do R?

Vamos l:

cdigos<-ifelse(dados$Ambiente=="Mata_Primria", "MP", "MS")
novos.nomes<-paste(row.names(dados), cdigos, sep="-")
amb.cluster$labels<-novos.nomes

plot(as.phylo(amb.cluster))

O que fizemos aqui?

  • ifelse voc j conhece, e foi usada para criar cdigos para serem acrescentados aos nomes;
  • paste permite que eu junte nomes, expresses e coisas assim, e foi usada para criarmos os nomes novos;
  • $labels foi usado para acessarmos os nomes usados no cluster.

Bacana, n?

Mas pera, como voc sabia desse esquema do $labels?

Existe uma funo bem bsica mas extremamente til no R, chamada ls.

Esta funo serve, inicialmente, para listarmos os objetos na memria do R. Por exemplo:

Mas ela tambm pode ser usada para listarmos o que est dentro de um objeto, o que pode nos ajudar a entender os seus componentes. No caso do cluster, eu soube que os nomes deveriam estar ali, por olhei para o resultado do ls, assim:

Escolhendo outras mtricas de distncia

Ao fazer o nosso cluster la em cima, trabalhamos com a distncia euclidiana. Mas, como vimos, outras mtricas podem ser necessrias, dependendo de coisas como a natureza dos meus dados e as minhas perguntas.

Vamos, ento, olhar rapidamente para alguns exemplos bem simples, ok? Voc ir precisar do pacote vegan.

Coeficiente de Bray-Curtis:

library(vegan)
## Loading required package: permute
## Loading required package: lattice
## This is vegan 2.0-10
dist.bray<-vegdist(dados[,7:40], method="bray")
cluster.bray<-hclust(dist.bray, method="average")
plot(cluster.bray, hang=-1)

Aqui usamos a funo vegdist, que tem um monte de mtricas acessveis pelo argumento method;

Coeficiente de Jaccard:

dist.jac<-vegdist(dados[,7:40], method="jac", binary=T)
cluster.jac<-hclust(dist.jac, method="average")
plot(cluster.jac, hang=-1)

Aqui adicionamos o argumento binary para indicar que queremos usar apenas a presena e ausncia das espcies.

Outras mtricas?

O pacote vegan possui uma funo bem bacana, que permite criar uma mtrica desejada. Vale pena dar uma espiada nela:

?designdist
